## 요약

| 관점 | 내용 |
|------|------|
| **문제** | Stellavault는 탄탄한 핵심 기술(하이브리드 검색, 3D 그래프, FSRS, MCP 12개 도구, 인텔리전스 패널)을 갖추고 있지만, 작동하는 프로토타입을 사람들이 채택하고 비용을 지불하는 제품으로 만드는 데 필요한 완성도, 안정성, "마지막 한 걸음" 기능이 부족하다. 주요 격차: 온보딩 마찰, 오프라인 복원력 부재, 플러그인 시스템 부재, 협업 기능 부재, 불완전한 그래프 상호작용, Pro/Cloud 가치 게이트 부재. |
| **해결책** | 세 가지 전략 축: (1) **완성도 & 안정성** -- 온보딩 위자드, 에러 복구, 성능 최적화, 접근성, i18n (2) **고급 인텔리전스** -- 지식의 시맨틱 버저닝, 모순 감지, AI 생성 학습 경로, 예측적 갭 분석, 컨텍스트 인식 MCP 라우팅 (3) **수익화 & 성장** -- 플러그인 아키텍처, 클라우드 동기화 백본, 팀 협업, 지식 팩 마켓플레이스, 임베딩 가능한 그래프 위젯 |
| **주요 기능/UX** | 온보딩 위자드, Plugin SDK, 지식 버저닝, AI 학습 경로 생성기, 그래프 임베드 위젯, 클라우드 동기화 엔진, 팀 볼트, 실시간 협업 인디케이터, 알림 센터, 키보드 기반 그래프 내비게이션 |
| **핵심 가치** | "개인 도구에서 지식 운영 체제로 -- Stellavault는 당신의 뇌, 노트, AI 에이전트 사이의 인텔리전스 레이어가 된다" |

### 1.3 기술 스택

- **런타임**: Node.js 20+, TypeScript, ESM
- **모노레포**: npm workspaces (core, cli, graph, sync)
- **벡터 DB**: SQLite + sqlite-vec
- **임베딩**: @xenova/transformers (로컬, API 키 불필요)
- **3D**: React Three Fiber + Three.js + Zustand
- **빌드**: Vite (graph), tsc (core/cli)
- **테스트**: Vitest (116개 테스트)

## 2. 디스커버리 분석 -- Opportunity Solution Tree

### 2.1 5단계 디스커버리 체인

#### 1단계: 브레인스토밍 -- Stellavault를 필수 도구로 만들 요소는?

타겟 페르소나(AI 활용 개발자, 500개 이상 노트 보유 지식 근로자)와의 인터뷰/관찰:

| # | 관찰 내용 | 출처 |
|---|-----------|------|
| O1 | "설치했는데 인덱싱 후 뭘 해야 할지 몰라서 포기했다" | 온보딩 마찰 |
| O2 | "3D 그래프는 멋지지만 키보드만으로는 탐색이 안 된다" | 접근성 격차 |
| O3 | "내 지식 그래프를 블로그/포트폴리오에 임베드하고 싶다" | 바이럴 유통 |
| O4 | "볼트에 노트가 5,000개 넘으면 느려진다" | 성능 한계 |
| O5 | "볼트가 3개(개인, 업무, 연구)인데 통합 검색이 안 된다" | 멀티 볼트 필요 |
| O6 | "팀이 접근 제어가 있는 지식 베이스를 공유하길 원한다" | 협업 |
| O7 | "감쇠 알림은 유용한데 목록이 아니라 학습 경로가 필요하다" | 인텔리전스 깊이 |
| O8 | "다른 노트에서 모순되는 내용을 쓰고도 몰랐다" | 일관성 검증 |
| O9 | "노트가 언제가 아니라 어떻게 변했는지 보고 싶다" | 시맨틱 버저닝 |
| O10 | "Zotero 논문을 추가하는 플러그인을 만들 수 있나?" | 확장성 |
| O11 | "MCP는 좋지만 Cursor/Windsurf에서도 작동해야 한다" | 멀티 클라이언트 MCP |
| O12 | "비행기에서 오프라인으로 쓰고 싶다" | 오프라인 복원력 |
| O13 | "CLI는 강력하지만 비개발자를 위한 웹 대시보드가 필요하다" | 웹 UI |
| O14 | "벡터 DB 클라우드 백업이 있다면 돈을 내겠다" | 수익화 신호 |
| O15 | "임베딩 실패 시 에러 메시지가 알아먹기 어렵다" | DX 완성도 |

## 9. 성장 루프

### 루프 1: 시각적 바이럴 (주요 성장 엔진)

```
사용자가 3D 지식 그래프를 탐색
    -> 스크린샷/녹화 촬영 (ExportPanel, 이미 구현)
    -> X/Twitter/LinkedIn/블로그에 공유
    -> "이 도구 뭐야?" 댓글
    -> stellavault.dev 워터마크 -> GitHub 방문
    -> 신규 사용자 설치 -> 자신만의 그래프 생성
    -> 자신의 그래프 공유 (반복)

가속기:
- F-A08 임베딩 가능 위젯: 사용자 웹사이트에 항상 노출되는 바이럴 면
- F-A09 지식 프로필: "powered by Stellavault"가 있는 공유 가능 URL
- F-A10 OG 이미지 생성: 소셜 공유를 위한 자동 미리보기
```

### 루프 2: MCP 네트워크 효과

```
개발자가 Stellavault를 MCP 서버로 설치
    -> Claude Code/Cursor에서 매일 12개 도구 사용
    -> 인텔리전스 도구 발견 (감쇠, 갭, 브리핑)
    -> "다른 MCP 서버보다 훨씬 낫다"
    -> 개발 커뮤니티 / MCP 레지스트리에서 추천
    -> 더 많은 MCP 사용자가 Stellavault 발견
    -> 플러그인 개발자가 가져오기 도구 제작 (Zotero, Readwise 등)
    -> 더 많은 데이터 소스 -> 더 많은 가치 -> 더 많은 사용자 (반복)
```

### 루프 3: 지식 인텔리전스 고착성

```
사용자가 볼트 인덱싱 -> 감쇠 알림 확인
    -> 잊힌 노트 리뷰 ("아, 이거 내가 썼었네!")
    -> 학습 경로 추천 받음
    -> 갭을 채우기 위해 새 노트 작성
    -> 지식 그래프가 더 풍부해짐
    -> 인텔리전스 기능이 더 가치 있어짐
    -> 사용자가 의존하게 됨 (긍정적 록인)
    -> 결국 투자 보호를 위해 클라우드 백업 Pro로 전환
    -> (반복 -- 인텔리전스 가치가 시간과 함께 복리로 증가)
```

### 루프 4: 팀 확장 (미래)

```
개인 개발자가 개인 지식용으로 Stellavault 사용
    -> 의사결정 저널 MCP 도구 발견
    -> "우리 팀에 이게 필요해" 깨달음
    -> 매니저에게 팀 볼트 제안
    -> 팀 채택 (Team 티어 수익)
    -> 신규 입사자가 팀 지식으로 온보딩
    -> 팀 지식 성장
    -> 더 많은 팀원이 필수 도구로 인식
    -> 회사 내 다른 팀도 채택 (반복)
```

## 11. 경쟁 배틀카드

### vs. MCPVault / Obsidian MCP 서버들

| 반론 | 대응 |
|------|------|
| "mcpvault가 설정이 더 간단하다" | Stellavault의 온보딩 위자드로 설정이 똑같이 쉬워지며, MCP 도구 12개 vs 5-7개, 3D 시각화, FSRS 인텔리전스, Knowledge Pack 내보내기까지 -- mcpvault는 절대 갖지 못할 기능들 |
| "AI에 기본 검색만 있으면 된다" | 기본 검색은 볼트가 500개 노트를 넘기 전까지만 괜찮음. Stellavault의 BM25+Cosine+RRF 하이브리드 검색은 30%+ 더 정확하고, 인텔리전스 기능(감쇠, 갭)이 "이미 이거 썼는데" 문제를 방지 |
| "mcpvault가 Obsidian REST API로 작동해서 편하다" | Stellavault는 .md 파일을 직접 인덱싱 -- Obsidian이 실행되지 않아도 작동. 지식이 항상 AI 에이전트에 접근 가능하며, 앱에 종속되지 않음 |

### vs. Khoj

| 반론 | 대응 |
|------|------|
| "Khoj가 더 많은 소스를 지원한다 (Notion, PDF, 이미지)" | Stellavault는 Obsidian 볼트에 특화되어 더 깊은 분석을 제공. 인텔리전스 레이어(FSRS 감쇠, 갭 탐지, 모순 탐지, 학습 경로)는 검색+채팅을 훨씬 넘어섬 |
| "Khoj에 자동화와 리서치 모드가 있다" | 철학이 다름: Khoj는 답을 검색하도록 돕고, Stellavault는 지식을 이해하도록 돕는다 -- 무엇을 잊고 있는지, 무엇이 모순되는지, 다음에 무엇을 배워야 하는지 |
| "Khoj에 모바일 앱이 있다" | 모바일은 로드맵에 있음 (Phase F). 현재 Stellavault의 MCP 서버는 어떤 기기의 어떤 AI 도구를 통해서든 지식에 접근 가능 |

### vs. Vestige

| 반론 | 대응 |
|------|------|
| "Vestige도 FSRS와 MCP를 쓴다" | 범위가 완전히 다름. Vestige는 AI 에이전트 세션 메모리(대화에서 일어난 일)를 추적하고, Stellavault는 개인 지식(노트, 문서, 결정)을 추적. 보완적 -- 둘 다 사용 가능 |
| "Vestige는 Rust로 작성돼서 더 빠르다" | Stellavault의 병목은 임베딩 연산(언어와 무관하게 동일 속도). 저장/검색은 SQLite-vec가 이미 네이티브에 가까운 속도. 인텔리전스 레이어(FSRS, 갭, 학습 경로)가 Vestige가 개인 지식에 제공하지 못하는 가치를 더함 |

### vs. InfraNodus

| 반론 | 대응 |
|------|------|
| "InfraNodus의 그래프 분석이 더 정교하다" | InfraNodus는 클라우드 전용($9-29/월)이며 MCP 통합이 없음. Stellavault는 프라이버시를 유지하며 무료로 로컬 실행, 12개 MCP 도구로 AI 에이전트 통합, 2D 네트워크 뷰보다 몰입적인 3D 시각화 |
| "InfraNodus에 이제 Obsidian 플러그인이 있다" | 그들의 플러그인은 분석을 위해 데이터를 클라우드로 전송. Stellavault는 모든 것을 로컬에 유지. 게다가 FSRS 감쇠, Knowledge Pack, 그리고 InfraNodus에 없는 전체 인텔리전스 세트 제공 |
