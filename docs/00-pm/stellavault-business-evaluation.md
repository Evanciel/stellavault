## 0. 전제: 현재 상태 정직하게 인식하기

**코드가 있다고 제품이 있는 것이 아닙니다.**

## Part I: 시장 현실 검증 (Discovery)

### 1.1 Obsidian + MCP 교차 시장은 얼마나 되나?

| 수치 | 추정 | 근거 |
|------|------|------|
| Obsidian 전체 사용자 | ~5M | 공식 발표 기반, 자사 추정 |
| MCP SDK 월간 다운로드 | 97M (2026.03) | Anthropic 공식, 전체 생태계 |
| MCP 서버를 직접 구성해 쓰는 개발자 | ~500K-1M | 97M 다운로드 중 실사용 추정 (CI/CD 중복, 자동화 제외) |
| Obsidian + MCP 동시 사용자 | ~50K-200K | 교차 추정. Obsidian MCP 서버가 60개+ 존재하나 각각 수천 다운로드 수준 |
| 이 중 CLI 도구를 쓰는 파워유저 | ~10K-30K | Obsidian 커뮤니티 포럼 MCP 관련 글 빈도 기반 |
| $12/월을 낼 의향이 있는 사용자 | ~1K-5K | 유사 도구(InfraNodus $9-29, Mem $8-10) 유료 전환율 기반 |

**결론: 비치헤드 시장이 극히 작다.**

"Obsidian을 쓰면서 MCP를 구성하고 CLI에 익숙한 개발자"는 전 세계에 만 명 내외다.
이 중 월 $12를 낼 사람은 기껏해야 수천 명이다.
이 시장만으로는 사업이 되지 않는다.

### 1.2 "지식 벡터화 + P2P 공유"에 돈을 낼 사람이 있나?

**솔직한 답: 거의 없다.**

| 질문 | 현실 |
|------|------|
| 임베딩 기반 검색이 Google/ChatGPT보다 나은가? | **아니다.** 500-5000개 노트에서 BM25+Vector가 더 정밀할 수 있지만, 대다수 사용자는 "그냥 ChatGPT에 물어봐도 되는데?"라고 생각한다 |
| P2P로 동료 vault를 검색하는 것이 Slack/이메일보다 나은가? | **대부분의 경우 아니다.** "야 그거 어디 정리했어?" 하고 물어보는 게 더 빠르다 |
| 임베딩만 공유한다는 게 안전한가? | **점점 아니다.** 2025-2026 연구에서 ALGEN이 1,000개 alignment sample로 ROUGE-L 45-50% 재구성. Zero2Text는 black-box에서도 작동. 384차원 all-MiniLM-L6-v2는 더 취약 |
| Federation 네트워크가 유용한 크기에 도달할 수 있나? | **극히 어렵다.** 100노드에서 가치가 생기는데, 100명의 파워유저가 Hyperswarm P2P 노드를 상시 가동할 동기가 없다 |

### 1.3 실제 경쟁자와의 정직한 비교

#### Stack Overflow / ChatGPT / Perplexity vs Stellavault

| 항목 | 그들 | Stellavault |
|------|------|-------------|
| **도달 가능한 지식 범위** | 인터넷 전체 | 내 노트 500-5000개 |
| **응답 속도** | 1-3초 | 1-3초 (동일) |
| **개인 컨텍스트** | 없음 | **있음** (유일한 우위) |
| **비용** | 무료-$20/월 | $12/월 + 설치/설정 노력 |
| **진입 장벽** | 없음 | CLI 설치, vault 인덱싱, MCP 설정 |

유일한 경쟁 우위는 "개인 컨텍스트"인데, 이것이 $12/월의 가치가 있으려면 **주 30분 이상의 시간을 절약**해야 한다. MCP 도구를 통해 AI가 내 과거 결정/노트를 참조하는 것은 분명히 가치가 있지만, 이 가치를 체감하려면:

1. 최소 500개 이상의 노트가 있어야 하고
2. AI 코딩 도구를 매일 사용해야 하고
3. MCP 설정을 완료해야 한다

이 세 조건을 모두 충족하는 사람이 타겟 시장이다. **매우 좁다.**

#### Notion AI vs Stellavault

| 항목 | Notion AI | Stellavault |
|------|-----------|-------------|
| 사용자 기반 | 수천만 | 0 |
| 설치 | 이미 쓰고 있음 | npm install + 설정 |
| AI 기능 | Q&A, 요약, 생성 | 검색, 그래프, FSRS, MCP |
| 프라이버시 | 클라우드 | 로컬 |
| 팀 기능 | 완전한 협업 | 코드만 있음 |
| 가격 | $10/월 (이미 쓰는 도구에 추가) | $12/월 (새 도구) |

**Notion AI가 이기는 이유:** 사람들은 이미 쓰는 도구에 AI를 추가하지, AI를 위해 새 도구를 설치하지 않는다.

#### Khoj vs Stellavault

| 항목 | Khoj | Stellavault |
|------|------|-------------|
| 소스 지원 | Obsidian, Notion, PDF, 이미지, 웹 | Obsidian only |
| 플랫폼 | 브라우저, 데스크톱, 폰, WhatsApp | CLI + 로컬 웹 |
| AI 모델 | GPT, Claude, Gemini, Llama, 로컬 | MCP (간접) |
| 자동화 | 예 (automations, scheduled agents) | 아니오 |
| 커뮤니티 | GitHub 21K+ stars | 0 stars |
| 가격 | 무료 (셀프호스팅) | 무료 (코어) |

**Khoj가 이기는 이유:** 더 넓은 소스 지원, 더 쉬운 접근성, 더 큰 커뮤니티. Stellavault의 기술적 깊이(FSRS, 3D 그래프, RRF)는 인상적이지만, 대부분의 사용자는 "기본 검색 + 채팅"이면 충분하다.

### 1.4 "있으면 좋은 것" vs "없으면 안 되는 것"

**냉정한 평가: "있으면 좋은 것"에 가깝다.**

| 기능 | 평가 | 이유 |
|------|------|------|
| MCP 기반 개인 지식 검색 | **있으면 상당히 좋음** | AI 코딩 시 과거 결정 참조는 실제로 유용. 하지만 없어도 코딩은 됨 |
| 3D 지식 그래프 | **있으면 멋짐** | 시각적 임팩트는 있으나 실질적 생산성 향상은 제한적. "와" 하고 한 번 보고 안 봄 |
| FSRS 감쇠 추적 | **있으면 좋음** | 노트가 1000개 넘는 사람에게만 의미. "내 노트를 잊고 있다"를 인식하는 것 자체가 새로운 문제 |
| Federation P2P | **있으면 좋음** | 기존에 없던 문제를 해결하려 함. "동료 vault 검색이 필요하다"고 느끼는 사람이 거의 없음 |
| 모순 탐지 | **있으면 좋음** | 자기 모순을 발견하는 건 재밌지만 필수 기능은 아님 |
| 학습 경로 | **있으면 좋음** | Anki/FSRS 기반 학습자에게는 가치 있으나 니치 |

**"없으면 안 되는 것"이 되려면:**
- "AI 코딩 어시스턴트가 내 노트를 자동으로 참조하여 이전에 내린 결정과 일관된 코드를 작성한다" -- 이것이 **주 1시간 이상** 절약된다는 것을 증명해야 함
- 현재는 이 증명이 없음. 사용자 0명이므로 데이터도 없음

### 1.5 실패할 가장 큰 이유 3가지

#### 이유 1: 솔로 개발자의 범위 초과 (Scope Creep Death)

현재 구현된/계획된 기능 목록:
- 인덱서, 검색엔진, 벡터 스토어, 3D 시각화, FSRS 감쇠, 갭 탐지, 중복 탐지, 모순 탐지, 학습 경로, 시맨틱 버저닝, 예측적 갭 분석, 알림 시스템, CLI 22개, MCP 13개, Pack 시스템, 마켓플레이스, Federation P2P (Hyperswarm), 트러스트/레퓨테이션, 프라이버시 레이어, 크레딧 시스템, 클라우드 싱크, 팀 볼트, RBAC, 플러그인 SDK, 웹훅, 대시보드, PWA, 음성 캡처, i18n, 멀티볼트...

**이것은 한 사람이 유지보수할 수 없는 범위다.**

2026년 오픈소스 메인테이너 위기 데이터:
- 오픈소스 메인테이너의 60%가 번아웃으로 프로젝트를 그만두거나 고려 중
- 거의 절반이 솔로 메인테이너
- Kubernetes Ingress NGINX는 메인테이너 번아웃으로 2026년 3월 이후 보안 패치 중단

**Stellavault는 이미 기능 범위에서 5-10명 팀 규모의 제품이다.** 솔로 개발자가 이 모든 것을 동시에 유지보수하는 것은 물리적으로 불가능하다.

#### 이유 2: 0에서 1로의 미증명 (Zero User Problem)

- 사용자가 없으므로 어떤 기능이 가치 있는지 알 수 없음
- PMF(Product-Market Fit)를 검증할 데이터가 없음
- PRD의 페르소나("Minjun", "Soyeon", "Jaehyun")는 추측일 뿐 실제 인터뷰가 아님
- 가격 탄력성, 전환율, 리텐션 -- 모든 핵심 비즈니스 지표가 가설 상태

**50개의 기능을 만드는 대신, 5명의 실제 사용자를 만드는 것이 100배 더 가치 있다.**

#### 이유 3: Federation은 해결책이 아닌 문제를 찾는 기술 (Solution in Search of a Problem)

Federation PRD는 기술적으로 인상적하다. Hyperswarm P2P, 임베딩 전용 공유, 차등 프라이버시, Web of Trust, 14개 CLI 커맨드, 11주 로드맵...

**하지만 누가 이것을 원하는가?**

"내 Obsidian 노트의 임베딩을 P2P로 동료에게 공유하고 싶다"고 말하는 사람을 실제로 만나본 적이 있는가?

Federation이 가치가 되려면:
1. 충분한 수의 Stellavault 단독 사용자가 먼저 있어야 하고 (현재 0)
2. 그들이 "다른 사람의 지식을 검색하고 싶다"고 실제로 요청해야 하고 (증거 없음)
3. 기술적 장벽(항시 가동 노드, NAT 통과, 임베딩 모델 호환)을 넘겨야 함

**Federation은 최소 2-3년 뒤에나 의미 있는 기능이다.** 지금 구현하는 것은 시간 낭비에 가깝다.

## Part II: 수익 현실성 (Strategy)

### 2.1 $12/월 Pro를 실제로 몇 명이 구매할까?

**현실적 추정 (Year 1):**

| 시나리오 | GitHub Stars | 월간 사용자 | Pro 전환율 | Pro 유료 사용자 | 월 매출 |
|----------|:-----------:|:-----------:|:----------:|:--------------:|:-------:|
| 최악 (현실적) | 200-500 | 50-100 | 2% | 1-2명 | $12-24 |
| 보통 | 1K-3K | 300-500 | 3% | 9-15명 | $108-180 |
| 낙관적 | 5K-10K | 1K-2K | 5% | 50-100명 | $600-1,200 |
| 대박 (Product Hunt #1) | 10K+ | 5K+ | 5% | 250+ | $3,000+ |

**비교 기준점:**
- InfraNodus: 유료 ($9-29/월), 추정 수백-수천 유료 사용자, 2014년부터 10년간 개발
- Obsidian Sync: $4/월, 추정 수만 유료 사용자, Obsidian의 직접 제품
- Mem.ai: $8-10/월, VC 펀딩 받고도 고전 중

**Year 1에 Pro 유료 사용자 100명을 달성하면 매우 성공적인 것이다.** 이 경우 월 $1,200, 연 $14,400. 서버/인프라 비용(R2, 도메인, CDN)을 겨우 충당하는 수준.

### 2.2 Enterprise Federation 납품 현실성

**솔직하게: 현재로서는 비현실적이다.**

| 장벽 | 설명 |
|------|------|
| **보안 감사** | 엔터프라이즈 고객은 보안 감사(SOC2, ISO27001)를 요구. 비용 $50K-200K, 솔로 개발자에게 불가능 |
| **SLA** | 99.9% 가동 시간 보장? 혼자서? |
| **지원** | 24/7 기술 지원? |
| **책임 보험** | 데이터 유출 시 책임은? |
| **구매 프로세스** | 엔터프라이즈 소프트웨어 구매에 6-18개월 소요 |
| **경쟁** | Glean ($300M+ 펀딩), Notion, Confluence가 이미 엔터프라이즈 지식 관리 |

"$20/user/mo Enterprise Federation"은 PRD 상의 숫자일 뿐이다. 실제로 어떤 회사가 개인 개발자의 P2P 지식 공유 도구에 사원 수십 명의 데이터를 맡기겠는가?

### 2.3 토큰/NFT 없이 지속 가능한 비즈니스 모델인가?

**가능하지만, 규모가 매우 작다.**

| 모델 | 현실성 | 예상 수익 |
|------|--------|----------|
| Open Core (Free/Pro) | 가능 | Year 1: $1K-15K ARR |
| Cloud Sync 구독 | 가능 (구현 필요) | 가변적 |
| Team Vault 구독 | 어려움 (경쟁 치열) | 미지수 |
| Knowledge Pack 마켓플레이스 | 매우 어려움 | ~$0 |
| 컨설팅/커스터마이제이션 | 가능하지만 확장 불가 | 시간당 |
| GitHub Sponsors | 가능 | $100-2,000/월 |

**핵심 문제:** Open Core 모델에서 무료 코어가 충분히 좋으면 Pro를 안 사고, 코어가 부족하면 사용자가 안 온다. 이 밸런스를 찾는 것이 매우 어렵다.

토큰/NFT는 인센티브 PRD에서 이미 올바르게 Phase 3 이후로 미루었다. 현재 시점에서 토큰은 사업 모델이 아니라 복잡성 추가일 뿐이다.

### 2.4 Break-even까지 얼마나 걸리나?

**가정:** 최소 비용으로 운영

| 비용 항목 | 월 비용 |
|-----------|---------|
| 도메인 + 호스팅 | $20 |
| R2/S3 (클라우드 싱크) | $10-50 |
| npm Pro (선택) | $0-7 |
| 총 고정 비용 | ~$30-80/월 |

| 시나리오 | Break-even 시점 | 조건 |
|----------|:---------------:|------|
| 최소 비용 ($30/월) | Pro 3명 ($36) | 가능하지만 무의미한 수준 |
| 서비스 비용 포함 ($200/월) | Pro 17명 | 6-12개월 |
| 시간 비용 포함 ($5K/월) | Pro 417명 | 2-3년 (낙관적) |
| 풀타임 환산 ($10K/월) | Pro 834명 | 3-5년 (비현실적?) |

**솔로 개발자의 시간을 비용으로 계산하면, 이 사업은 3-5년 내에 Break-even이 어렵다.** 사이드 프로젝트나 학습 목적이라면 괜찮지만, 생계 수단으로는 부적합하다.

### 2.5 VC 투자 가능성

**솔직한 평가: 현재 상태로는 시드 투자를 받기 매우 어렵다.**

| VC가 보는 것 | Stellavault 상태 | 평가 |
|-------------|-----------------|------|
| 팀 | 솔로 개발자 | **부정적.** VC는 최소 2-3명 팀을 원함 |
| 트랙션 | 사용자 0, 매출 $0 | **치명적.** 시드도 최소한의 트랙션을 원함 |
| 시장 크기 | PKM 니치 | **제한적.** TAM이 충분히 크지 않음 |

### 3.2 Stellavault의 진짜 경쟁 우위 (있다면)

**솔직하게 존재하는 것:**

1. **MCP 도구 수와 깊이** -- 13개 도구는 Obsidian MCP 서버 중 가장 많고 다양함. 단순 CRUD가 아닌 decay, gaps, brief, learning-path 등 지능형 도구
2. **FSRS 기반 지식 감쇠** -- 학습 과학을 노트 관리에 적용한 유일한 도구 (Vestige 제외, 하지만 Vestige는 에이전트 메모리이지 개인 지식이 아님)
3. **완전 로컬** -- 임베딩도 @xenova/transformers로 로컬 생성. API 키 불필요. 프라이버시 민감한 사용자에게 진짜 우위

**솔직하게 경쟁 우위가 아닌 것:**

1. **3D 그래프** -- "와" 하고 한 번 보고 끝. 일상적 사용에서 2D 그래프보다 나은 점이 없음. 오히려 성능 부담
2. **Federation** -- 사용자가 없으므로 네트워크도 없음. 네트워크 없는 P2P는 가치가 0
3. **"knowledge intelligence platform"이라는 카테고리** -- 카테고리 생성은 시장이 존재해야 의미 있음
4. **RRF 하이브리드 검색의 30%+ NDCG 우위** -- 대부분의 사용자는 검색 품질 차이를 체감하지 못함

### 3.3 "오픈소스 + Federation"이 경쟁 우위가 되는 실제 사례

| 사례 | 결과 | 교훈 |
|------|------|------|
| **Mastodon** (오픈소스 + Federation) | 1000만+ 사용자, 하지만 수익 모델 부재. Gargron(메인 개발자) Patreon 의존 | Federation은 성장을 만들지만 돈을 안 벌어줌 |
| **Matrix/Element** (오픈소스 + Federation) | 기업 납품으로 수익화. 정부/군 사용 | B2B로 전환해야 수익화 가능 |
| **PeerTube** (오픈소스 + Federation) | 소규모 커뮤니티. YouTube와 경쟁 불가 | 대형 플랫폼의 네트워크 효과를 이길 수 없음 |
| **Bluesky** (분산 프로토콜) | 4000만+ 사용자, VC 펀딩 $150M+ | 성공했지만 실질적으로 중앙화됨 |

**교훈:**
- 오픈소스는 채택을 돕지만 수익화를 어렵게 함
- Federation은 이상적이지만 "충분히 좋은" 중앙화 솔루션을 이기기 어려움
- 수익화에 성공한 경우는 대부분 B2B (Matrix/Element의 정부/기업 납품)

### 3.4 솔로 개발자가 이 규모를 유지할 수 있나?

**아니다.**

현재 코드베이스 규모:
- core: 65+ 파일, 6,839줄
- cli: 22 커맨드, 3,458줄
- graph: 20 컴포넌트, 수천 줄
- sync: 별도 패키지
- **총 ~15,000-20,000줄 TypeScript**

이것은 이미 한 사람이 "모든 것을 머릿속에 담을 수 있는" 한계를 넘었다. 여기에 Federation(11개 파일), Cloud, Team, Plugin, Marketplace까지 있다.

**유지보수 현실:**
- 보안 취약점 패치 (npm 의존성만 수십 개)
- Node.js/npm/Vite 업그레이드
- MCP 프로토콜 변경 대응
- 사용자 버그 리포트 대응
- 문서 작성/유지
- 새 기능 개발

한 사람이 이 모든 것을 하면서 동시에 사업도 운영하는 것은 **6-12개월 내에 번아웃**으로 이어질 가능성이 높다.

| 수익화 가능성 | **2/10** | Year 1 매출 $1K-15K 예상. 풀타임 급여 수준까지 3-5년. 솔로로는 Break-even 어려움 |
| 실행 가능성 (솔로) | **3/10** | 현재 코드베이스는 이미 한 사람이 감당하기 어려운 규모. 마케팅/지원/개발 동시 수행 불가능 |
| 투자 유치 가능성 | **2/10** | 사용자 0, 매출 $0, 솔로 개발자. 시드 투자의 최소 조건을 충족하지 못함 |
| **종합** | **3.5/10** | 기술은 인상적이나, 사업으로서의 기반이 전무. "멋진 사이드 프로젝트"와 "사업" 사이의 간극이 큼 |

### 5.3 Go/No-Go 판단

#### 현재 방향대로의 사업: **No-Go**

"Stellavault + Stella Network를 SaaS 사업으로 키워서 풀타임으로 먹고살겠다"는 현재 상태로는 **비현실적**이다.

**이유:**
1. 시장이 너무 작다 (Obsidian + MCP + CLI 파워유저)
2. 경쟁자가 너무 많고 자원이 압도적으로 부족하다 (Notion: $10B, Khoj: 21K stars + 팀)
3. Federation은 "해결책이 문제를 찾는" 상황이다
4. 솔로 개발자로 이 규모의 코드베이스를 유지하면서 사업도 하는 것은 불가능하다
5. 기술이 아무리 좋아도, 사용자가 0이면 사업이 아니다

#### 사이드 프로젝트/포트폴리오로서: **Go**

그러나 Stellavault는 **매우 인상적인 기술 포트폴리오 프로젝트**다.

| 자산 | 가치 |
|------|------|
| 벡터 검색 + 하이브리드 RRF | 시니어 엔지니어 수준 검색 구현 역량 증명 |
| MCP 서버 13개 | MCP 생태계 초기 전문가 |
| P2P Hyperswarm | 분산 시스템 구현 능력 |
| 3D R3F 시각화 | 프론트엔드 고급 역량 |
| FSRS 구현 | 학습 과학 + 소프트웨어 융합 |
| 모노레포 + 테스트 116개 | 소프트웨어 엔지니어링 성숙도 |

**이 포트폴리오 자체가 취업 시장에서 매우 강력한 시그널이다.**

#### 진행한다면 어떻게: **Lean + Focused**

만약 그래도 사업으로 진행하겠다면:

1. **기능의 80%를 잘라라.** Federation, Team, Cloud, Plugin, Marketplace, 인센티브 전부 동결
2. **"MCP 지식 서버" 하나에 집중하라.** "Obsidian vault를 AI에 연결하는 가장 쉬운 방법"
3. **3개월 안에 무료 사용자 100명을 만들어라.** 이것이 안 되면 시장이 없는 것
4. **사용자와 대화하라.** PRD를 쓰는 시간의 절반을 사용자 인터뷰에 쓰라
5. **혼자 하지 마라.** 공동 창업자 또는 핵심 기여자를 찾아라

### 5.4 피봇 제안: 같은 기술로 더 현실적인 사업

#### 피봇 A: "MCP Knowledge Server as a Service" (가장 현실적)

**아이디어:** Stellavault를 분해하여, MCP 서버 부분만 npm 패키지로 배포. "npm install stellavault && stellavault serve" 한 줄로 Obsidian/마크다운 vault를 MCP 서버로 변환.

| 항목 | 내용 |
|------|------|
| 타겟 | Claude Code/Cursor 사용자 중 Obsidian 사용자 (수만-수십만) |
| 차별화 | 설치 가장 쉬움 + 검색 가장 정확 + 도구 가장 많음 |
| 수익 모델 | 기본 무료, Pro (고급 도구 + 클라우드 백업) $5/월 |
| 장점 | 시장이 더 넓음, 설치 쉬움, Federation 불필요 |
| 리스크 | MCPVault 등과 직접 경쟁 |

#### 피봇 B: "AI Code Reviewer with Personal Context" (높은 가치)

**아이디어:** MCP + 개인 지식을 활용한 AI 코드 리뷰 도구. "이 PR이 당신의 과거 아키텍처 결정과 일치하는지" 자동 검증.

| 항목 | 내용 |
|------|------|
| 타겟 | 개발 팀 (5-50명), 시니어 개발자 |
| 차별화 | 팀의 축적된 결정/패턴 기반 리뷰 (일반 AI 리뷰와 다름) |
| 수익 모델 | $10-30/user/month |
| 장점 | B2B는 지불 의향 높음, 명확한 ROI (리뷰 시간 절감) |
| 리스크 | GitHub Copilot 리뷰, CodeRabbit 등 경쟁 치열 |

#### 피봇 C: "Knowledge Decay Dashboard" (니치 but 독특)

**아이디어:** FSRS 감쇠 + 갭 탐지를 노트 앱 독립적인 SaaS로. 마크다운 파일 업로드 -> "당신이 잊고 있는 것" 대시보드.

| 항목 | 내용 |
|------|------|
| 타겟 | 의학생, 법대생, 자격증 준비생, 연구자 |
| 차별화 | Anki는 카드 단위, 이것은 노트 단위 감쇠 추적 |
| 수익 모델 | Freemium, $5-10/월 |
| 장점 | 학습자 시장은 지불 의향 높음, FSRS는 검증된 알고리즘 |
| 리스크 | 시장이 작을 수 있음, RemNote/Anki 생태계와 경쟁 |

#### 피봇 D: "3D Knowledge Graph Widget" (바이럴 잠재력)

**아이디어:** 3D 그래프만 떼어내어 임베드 가능한 위젯으로. 블로그/포트폴리오에 "내 지식 우주"를 보여주는 인터랙티브 그래프.

| 항목 | 내용 |
|------|------|
| 타겟 | 개발자 블로거, 테크 인플루언서, 이력서 사이트 |
| 차별화 | 시각적 임팩트 극대. "와" 팩터 |
| 수익 모델 | 기본 무료 (워터마크), Pro $3/월 (커스텀, 워터마크 제거) |
| 장점 | 바이럴 마케팅 자체가 제품, 개발 범위 좁음 |
| 리스크 | 사용자가 한 번 설정하고 취소할 수 있음 (LTV 낮음) |

#### 피봇 추천 순위

1. **피봇 A** (MCP Knowledge Server) -- 가장 현실적이고 기존 코드 활용 극대화
2. **피봇 B** (AI Code Reviewer) -- 가장 높은 수익 잠재력, 하지만 추가 개발 필요
3. **피봇 C** (Knowledge Decay Dashboard) -- 독특한 포지셔닝, 니치 시장
4. **피봇 D** (3D Graph Widget) -- 바이럴 잠재력, 하지만 수익 제한적

| 질문 | 답 |
|------|---|
| Stellavault는 좋은 기술인가? | **예.** 인상적인 구현. 포트폴리오로서 탁월 |
| Stellavault는 좋은 사업인가? | **아니오.** 시장 작음, 경쟁 치열, 솔로 한계 |

| 인센티브 설계는 좋은가? | **이론적으로 훌륭하지만 시기상조.** 네트워크가 없는데 인센티브를 설계하는 것은 허공에 집 짓는 것 |

| 가장 현실적인 다음 행동은? | **기능 80% 잘라내고, MVP 출시하고, 100명 무료 사용자를 먼저 만들어라** |
